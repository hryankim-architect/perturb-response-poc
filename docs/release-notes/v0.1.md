# v0.1, initial capability portrait

**Type:** new repository (capability portrait #10, the perturbation-biology axis)

## What this is

A clean-room POC of single-cell perturbation-response analysis (Perturb-seq
style) on synthetic data: per-perturbation effect recovery (differential
expression vs control) and held-out perturbation response prediction (ridge
feature→response with leave-out-perturbation CV). It completes the
single-cell / spatial / perturbation trio alongside `sc-tumor-annotator` (#7)
and `spatial-niche-mapper` (#9).

## Shipped in v0.1

- `synth.py`, deterministic Perturb-seq screen: control + perturbed cells, a
  per-perturbation feature embedding, and ground-truth sparse response programs
  derived linearly from the features (learnable cross-perturbation structure).
- `effect.py`, mean shift vs control: program recovery F1 + effect-size ranking
  Spearman vs ground truth.
- `predict.py`, ridge feature→response, leave-out-perturbation CV, with a
  mean-response baseline for comparison.
- `pipeline.py`, CLI wiring generate → effect recovery → held-out prediction,
  with audit + MLflow hooks; writes a metrics artifact.
- Substrate hooks (`audit`, `tracking`, `canary`), CI (ruff + pytest +
  scope-preamble + canary + English-only), MIT license, Makefile.

## Demo metrics (synthetic, seed 0; 40 perturbations, 200 genes)

| Metric | Value |
|---|---|
| Program recovery F1 | 0.989 |
| Effect-ranking Spearman | 0.986 |
| Held-out response correlation (ridge) | 0.598 |
| Held-out response correlation (mean baseline) | 0.045 |
| Feature-signal lift | +0.553 |

13 tests, ruff + English-only green; canary asserts program recovery above a
floor F1.

## Scope

Synthetic only; a transparent linear baseline, not a deep model; the held-out
0.60 correlation is a modest honest number for a hard task. See
`what-is-out-of-scope.md`.
